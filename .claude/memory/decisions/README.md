# Architectural Decisions

This directory stores important architectural and implementation decisions.

## Files

- `architecture.md` - Project structure decisions
- `dependencies.md` - Package and dependency choices
- `patterns.md` - Code pattern decisions
- `conventions.md` - Naming and style conventions

## Decision Entry Format

```markdown
## Decision: [Title]
**Date**: YYYY-MM-DD
**Status**: proposed | accepted | deprecated | superseded

### Context
[what prompted this decision]

### Decision
[what was decided]

### Rationale
[why this decision was made]

### Alternatives Considered
1. [Alternative 1] - Rejected because [reason]
2. [Alternative 2] - Rejected because [reason]

### Consequences
**Positive:**
- [benefit]

**Negative:**
- [drawback]

### Review Date
[when to reconsider]
```

## Usage

Check decisions before making architectural choices to ensure consistency.
