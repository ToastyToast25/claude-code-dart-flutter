# Session Logs

This directory stores session-specific learnings and notes.

## Naming Convention

Files should be named: `YYYY-MM-DD-brief-summary.md`

Example: `2026-01-21-initial-setup.md`

## Session Entry Format

```markdown
# Session: YYYY-MM-DD - [Brief Summary]

## Focus
[main goal of the session]

## Tasks Completed
- [task 1]
- [task 2]

## Errors Encountered & Fixed
### Error 1: [description]
- Fix: [solution]
- Added to memory: yes/no

## Decisions Made
- [decision 1]
- [decision 2]

## Patterns Discovered
- [pattern]

## Unresolved Issues
- [issue]

## Notes for Next Session
- [note]
```

## Usage

Create a session file at the end of significant work sessions to preserve context.
