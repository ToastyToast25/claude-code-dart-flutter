# Project Learnings

This directory stores learnings specific to this project's development.

## Naming Convention

Files should be named after the feature or area: `{feature-name}.md`

Example: `authentication.md`, `api-client.md`, `deployment.md`

## Project Learning Format

```markdown
# Learnings: {Feature/Area Name}

**Created**: YYYY-MM-DD
**Last Updated**: YYYY-MM-DD

## Overview
[brief description of this area]

## Key Decisions Made
- [decision 1] - [rationale]
- [decision 2] - [rationale]

## Implementation Notes
[important implementation details]

## Gotchas & Pitfalls
- [thing that tripped us up]
- [non-obvious behavior]

## Working Patterns
```dart
// Pattern that works well
[code example]
```

## Failed Approaches
```dart
// What we tried that didn't work
[code example]

// Why it failed
[explanation]
```

## Testing Notes
- [testing considerations]
- [mocking requirements]

## Performance Considerations
- [optimization notes]

## Future Improvements
- [planned enhancement]
```

## Usage

As features are developed, capture learnings here to help future development.
